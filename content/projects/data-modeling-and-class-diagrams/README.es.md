## Tu reto

En este proyecto vas a practicar **modelado de datos** y **diagramas de clases** para representar sistemas reales de forma clara antes de implementarlos.

Trabajaras con la herramienta [diagram.4geeks.com](https://diagram.4geeks.com/) en lugar de otras plataformas externas. El objetivo es que puedas identificar entidades, propiedades, tipos de datos y relaciones entre modelos.

Debes completar dos ejercicios y entregar los resultados exportados en PNG.

---

## Como iniciar el proyecto

1. Abre [diagram.4geeks.com](https://diagram.4geeks.com/).
2. Crea un diagrama nuevo para cada ejercicio.
3. Modela las entidades, sus propiedades con tipos de datos y las relaciones entre ellas.
4. Exporta cada resultado en formato PNG.

---

## Ejercicio 1: Reproductor de playlist de musica

Modela un sistema similar a Spotify, donde las canciones son objetos individuales que viven dentro de una lista de reproduccion.

Requisitos:

- Debe tener al menos 5 modelos
- Cada propiedad debe incluir su tipo de dato
- Debe indicarse que modelos se relacionan entre si
- El resultado final debe exportarse en PNG

Sugerencia: piensa en modelos como `User`, `Playlist`, `Song`, `Artist`, `Album` o cualquier otra variante coherente.

---

## Ejercicio 2: Billetera digital

Modela una billetera digital similar a Wise, donde exista un historial de movimientos.

Requisitos:

- Debe tener al menos 3 entidades
- Cada propiedad debe incluir su tipo de dato
- Debe indicarse que modelos se relacionan entre si
- El resultado final debe exportarse en PNG

Sugerencia: piensa en modelos como `Wallet`, `Account`, `Transaction`, `Currency`, `User` o variantes equivalentes.

---

## Que debes hacer

- [ ] Crear un diagrama para el reproductor de playlist
- [ ] Crear un diagrama para la billetera digital
- [ ] Usar tipos de datos en todas las propiedades
- [ ] Definir las relaciones entre modelos de forma clara
- [ ] Exportar ambos diagramas en PNG

---

## Que vamos a evaluar

- [ ] Claridad del modelado
- [ ] Correcto uso de entidades y relaciones
- [ ] Presencia de tipos de datos en las propiedades
- [ ] Cumplimiento del minimo de modelos o entidades solicitadas
- [ ] Coherencia general del diagrama

---

## Como entregar este proyecto

Debes entregar **dos imagenes PNG**:

- `music-playlist-class-diagram.png`
- `digital-wallet-class-diagram.png`

La configuracion de entrega espera **2 archivos PNG**.
